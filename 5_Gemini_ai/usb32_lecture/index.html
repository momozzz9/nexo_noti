01. Specification Evolution & Architecture

통합된 USB 3.2 에코시스템

USB 3.2 Rev 1.1은 기존의 USB 3.0(Gen 1) 및 USB 3.1(Gen 2)을 단일 사양으로 통합하고, Multi-lane 동작을 위한 물리적/논리적 계층 확장을 정의합니다.

  • USB 3.2 Gen 1x1: 5Gbps, 8b/10b 인코딩 (Legacy SuperSpeed)
  • USB 3.2 Gen 2x1: 10Gbps, 128b/132b 인코딩 (Legacy SuperSpeed+)
  • USB 3.2 Gen 1x2: 10Gbps (5Gbps x 2 Lanes), 8b/10b 인코딩
  • USB 3.2 Gen 2x2: 20Gbps (10Gbps x 2 Lanes), 128b/132b 인코딩

Key Architecture Update

물리 계층(PHY)의 데이터 전송률을 높이는 대신, Type-C의 가용 리소스를 병렬화하여 대역폭을 2배로 확장하는 전략을 취함. 이는 기존 Gen 2 PHY 설계 자산을 재활용하면서도 20Gbps를 달성할 수 있게 함.

두 개의 물리적 Lane을 논리적으로 결합하여 단일 고속 데이터 파이프라인을 형성하는 핵심 기술

Technical Deep Dive

  • Data Striping: 바이트 단위로 Lane 0과 Lane 1에 교차 분산 전송
  • Symbol Alignment: 수신단에서 TS1/TS2 Ordered Sets를 사용하여 두 Lane 간의 상징적 정렬(Symbol Lock) 수행
  • Throughput: Gen 2x2 기준, 이론적 대역폭 20Gbps 중 약 19.4Gbps의 유효 데이터 처리량 확보 (128b/132b 효율성 덕분)

04. Type-C CC State Machine & Power

Connection State Machine

  • Unattached: Rd(Device) 또는 Rp(Host) 상태 대기
  • Attach Detection: CC 핀 전압 강하 감지 (vRd/vRp)
  • Orientation Detection: CC1/CC2 중 활성 핀 식별 (Flip 판단)
  • VCONN Switching: E-marker 케이블 전원 공급 및 PD 메시징 시작
  • Dual-Lane Entry: PD VDM(Vendor Defined Message) 협상을 통한 x2 모드 활성화
HW Design Critical

Power Integrity: 20Gbps 동작 시 PHY의 소모 전력이 급증함. 특히 VDDA_1V8, VDD_0V9 라인의 Ripple 제어가 Jitter 성능에 결정적임.

PD Controller: x2 모드를 위해 CC 컨트롤러는 반드시 USB 3.2 x2 Link Training을 트리거할 수 있는 능력을 갖춰야 함.

05. Lane-to-Lane Skew & De-skew Logic

병렬 전송 시 두 Lane의 물리적 길이 차이로 인한 데이터 도달 시간차(Skew)는 시스템 안정성의 핵심 변수입니다.

Max Allowable Skew

6,400 ps (6.4ns)

Elastic Buffer Depth

수신단 PHY 내부에서 Skew를 흡수하기 위한 가변 깊이 버퍼

HW 설계 시 Skew 유발 요인

  • PCB Trace Length: Lane 0과 Lane 1 간의 패턴 길이 편차 (Matching 0.25mm 이내 권장)
  • Material Variance: FR4 재질의 불균일한 유전율(Dk) 분포로 인한 전파 속도 차이
  • Connector/Via: 각 Lane이 거치는 Via 개수 및 커넥터 핀 구조의 비대칭성

Recovery & Error Handling

링크 중 에러(LCRC/ECRC 실패) 감지 시 자동으로 Recovery 상태로 진입. 여기서 재훈련에 실패할 경우 하위 속도(Gen 2 -> Gen 1)로 강제 다운그레이드 수행.

07. Power Management States & Latencies

State Description Typical Exit Latency Power Saving
U1 Fast Idle (Clock Gating) < 10 µs Low
U2 Slow Idle (Partial PHY Off) ~ 100 µs Medium

LFPS (Low Frequency Periodic Signaling)

저전력 상태에서 링크를 깨우기 위한 신호(10~50MHz). HW 설계 시 LFPS 신호의 진폭(Amplitude) 및 듀티 사이클이 규격을 만족하지 못하면 Wake-up 실패의 원인이 됨.

08. Signal Integrity & PCB Layout Guide

Impedance & Loss

  • Differential Impedance: 90Ω ~ 100Ω (±10%)
  • Insertion Loss: < 6dB @ 5GHz (Nyquist for 10Gbps)
  • Use Low-loss dielectric (e.g., Megtron 6) for long traces

Routing Rules

  • P/N Length Matching: < 0.1mm (Intra-pair)
  • Lane-to-Lane Matching: < 0.25mm (Inter-pair)
  • Ground Stitching: Via 전환 시 반드시 GND Via 인접 배치
  • Back-drilling: 커넥터 및 Via의 Stub 제거 필수

Crosstalk Mitigation: TX와 RX 쌍 사이의 간격을 트레이스 폭의 5~6배 이상 확보하여 Near-end Crosstalk(NEXT) 차단 필요.

HW 설계 최종 체크리스트 (Samsung Focus)

  • Redriver/Retimer 필요성 검토 (Trace > 4-inch)
